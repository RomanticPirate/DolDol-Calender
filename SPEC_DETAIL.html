캘린더 렌더링

3.1 구조

7열 CSS Grid (.cal-grid) 안에 42개 셀 (.cal-cell). 각 셀은 날짜 숫자 + 공휴일명 + 일정 라벨 영역을 포함.

셀 높이95px 고정 (CSS height:95px). 일정 유무 무관
행 수항상 6행 (42셀). 부족분은 이전/다음 월 패딩
일정 최대셀 내 4개까지 라벨 표시. 초과분 "+N"
선택일셀 전체에 border:2px solid var(--p) + 연보라 배경
라벨 형태카테고리아이콘 + 색상배경 + 제목텍스트

3.2 렌더 프로세스

renderCal()→ 월 타이틀 갱신→ 42셀 생성→ 각 셀: 날짜+공휴일+일정라벨(최대4)+"+N"→ grid에 append→ renderDP() 패널

프로세스 흐름

15.1 앱 부팅

load()→User 있음?→YES: init()→renderCal + fetchH
User 없음?→온보딩→닉네임→프로필+캘린더+카테고리3+샘플2 생성→init()

15.2 일정 생성

FAB(+)→모달 (scrollTop=0)→입력→[저장]→검증→EVTS.push→히스토리→save(백업)→모달닫힘→renderCal→토스트

15.3 일정 삭제

[삭제]→confirm→deleteLog에 스냅샷→EVTS에서 제거→히스토리→save→모달닫힘→renderCal

15.4 닉네임 변경

설정→닉네임→팝업→입력→저장→U.name 변경→CAL.members 갱신→모든 댓글.name 갱신→save→renderStg+renderCal

15.5 .ics 익스포트

설정→내보내기→EVTS 순회→VEVENT 변환→RRULE+VALARM 매핑→Blob→download
